### 2. 覆盖率口径需要对客户讲清楚

线上 `/api/coverage` 的覆盖率口径是：

```text
覆盖率 = min(当前抓取量, 目标量) / 目标量
```

其中：

- 当前抓取量优先取 `fetched_urls`，即目标 URL 的抓取记录，包含成功抓取和部分失败状态；
- 如果没有 `fetched_urls`，回退到商品落库 URL 数；
- 目标量优先取工作区配置的目标 SKU，其次取 sitemap 产品 URL 总量，再回退到系统估算值；
- 展示时覆盖率会钳制在 100%，避免因 sitemap 变动或重复 URL 导致超过 100%。

因此覆盖率主要回答“站点 URL 是否跑到位”，不直接等价于“每个字段都完整”。字段完整性要看数据质量页中的价格、促销、销售、评论、流量和转化率信号。

## 四、数据抓取逻辑说明

### 1. 商品发现逻辑

系统按站点技术栈采用不同发现方式：

| 技术栈 / 平台 | 适用站点 | 抓取入口 |
|---|---|---|
| Shopify | SONGMICS、Yaheetech、BCP 等 | 商品 API、集合页、sitemap |
| Magento / Shopware / Shoper | Costway 欧洲站、Woltu 等 | sitemap、分类页、PDP 结构化数据 |
| Nuxt / SSR | Homary | 服务端渲染数据、页面内嵌 JSON、PDP |
| Vidaxl 自建站 | Vidaxl 欧洲站 | sitemap / category product URL / PDP |
| Flexispot 自定义前端 | Flexispot | 分类页与产品页解析 |

发现流程为：

1. 从 sitemap、分类页或平台 API 发现候选商品 URL；
2. 按站点规则过滤非商品 URL，例如 Vidaxl 只保留 `/p/` 产品路径；
3. 对已抓取 URL 去重，避免同一轮反复抓取；
4. 将新 URL 入队，由 worker 按站点并发和代理策略抓取；
5. 抓取结果写入商品表、价格历史、促销表、任务日志和覆盖率统计。

### 2. 商品字段解析逻辑

商品页解析优先级为：

1. 结构化 JSON / JSON-LD / 平台 API；
2. 页面内嵌状态数据；
3. DOM selector 兜底；
4. 无法解析时记录失败原因，不硬填假数据。

标准字段包括：`sku`、`spu`、标题、描述、图片、类目路径、售价、原价、币种、变体、评分、评论数、状态、库存、视频标识、免运费标识、标签、商品 URL、品牌、采集时间等。

### 3. 价格与促销逻辑

价格逻辑：

- 当前售价来自 PDP、列表页或平台 API；
- 原价 / compare price 存在且高于售价时，生成折扣信号；
- 每次采集写入价格历史，用于后续价格曲线。

促销逻辑：

- 价格下降、原价与售价差异、coupon/discount/bundle/free-shipping 等标签都会进入促销识别；
- 同一 SKU 多次采集后，系统可判断活动前价、活动中价和活动恢复；
- 促销开始/结束时间只有在页面明确展示倒计时或活动时间时才直接抓取，否则以检测时间作为发现时间。

### 4. 销量与营收估算逻辑

原始需求中的销量不是网页直接给出的真实销售额，系统采用估算逻辑：

```text
评论增量 = 本期评论数 - 上期评论数
预估销量 = 评论增量 / 留评率
预估营收 = 预估销量 × 当前售价
```

当前默认留评率配置为 2.5%。该逻辑必须满足两个前提：

- 同一 SKU 至少有两次以上稳定评论数历史；
- 商品价格字段稳定可用。

因此，在历史评论不足的站点上，销售和营收只能标注为“预估能力已具备，当前数据不足以作为验收闭环”。

### 5. 流量与转化率逻辑

流量和转化率不建议承诺由爬虫直接抓取。可交付方式应为：

- SimilarWeb / GA / BI / 客户内部文件导入；
- 系统按 `site + date` 存储并展示；
- 导入后与 SKU、价格、促销、评论趋势联动分析。

## 五、建议给客户的回复口径

建议对外表达为：

1. **基础商品数据已大幅补齐**：原始 46 个标杆站中 44 个已有线上数据，P0 的 20 个站点均有数据；整体覆盖率按线上覆盖口径为 99.54%。
2. **Vidaxl US/CA 需要专项处理**：这两个站点当前仍为 0，建议客户配合提供 Vidaxl B2B API 凭据；凭据到位后可按 API 路径补齐，比继续 storefront 抓取更稳定。
3. **字段完整性需分层验收**：商品基础字段和价格字段可以进入验收；促销字段需继续补站点规则；销量/营收需历史评论沉淀；流量/转化率需外部数据导入。
4. **抓取逻辑已明确**：商品发现、字段解析、价格促销、销量估算、覆盖率计算均有明确规则，后续验收应按这些口径逐项确认，避免把 sitemap URL 覆盖、SKU 落库、销售估算混为一个指标。
