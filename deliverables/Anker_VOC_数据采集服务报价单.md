| 费用类型 | 计费方式 | 说明 |

| 基础服务费 | 固定月费 | 当前量级资源池预估为 38,000 元/月，覆盖采集算力池、代理池、Amazon 账号、存储、采集运维和监控告警 |
| 可选增值项 | 按需启用 | P0 预留并发包、一次性接口适配开发、额外代理或账号成本 |

## 三、采集流量报价

### 3.1 报价依据

本报价以双方既有结算口径为基础，并按新需求中的优先级、登录态、SLA 和账号池占用差异进行拆分。

| 费用项 | 报价依据 |
|---|---|
| P1 登录态页面 3.30 元 / 1000 页面 | 沿用历史结算单中登录页面计费单价 |
| P1 非登录态页面 2.20 元 / 1000 页面 | 沿用历史结算单中非登录页面计费单价 |
| P0 紧急任务 | 在 P1 基础上按约 1.6 倍计价，用于覆盖 4 小时返回、抢占式调度、账号池优先占用和人工监控成本 |
| explore 探索任务 | 仅限非登录态公开页面探索，不占用登录账号池，不承诺全量和强 SLA，因此低于 P1 非登录态价格 |
| 评论数量计费 | 作为评论采集的可选口径，按成功返回评论条数计费；单价由页面计费口径折算，避免评论少但页面翻取多时客户成本过高 |
| 基础服务费 38,000 元/月 | 按当前 Anker 月度量级进行资源池预估，覆盖采集所需基础资源和服务，包括采集算力池、代理池、Amazon 账号、数据存储、Rufus / 特殊页面维护、风控变化处理、月度运维和月度对账 |

历史 P1 单价换算如下：

```text
登录态页面单价 = 3.30 元 / 1000 页面
非登录态页面单价 = 2.20 元 / 1000 页面
```

当前月度测算中的采集流量费计算如下：

```text
登录态页面费用 = 6,794,428 / 1000 * 3.30 = 22,421.61 元
非登录态页面费用 = 10,128,855 / 1000 * 2.20 = 22,283.48 元
采集流量费合计 = 44,705.09 元
月度预估合计 = 44,705.09 + 38,000.00 = 82,705.09 元
```

### 3.2 按页面量计费

计费单位：元 / 1000 成功页面。

| 优先级 | 登录态页面 | 非登录态页面 | SLA / 说明 |
|---|---:|---:|---|
| P0 紧急任务 | 5.30 | 3.60 | 抢占式调度，4 小时内返回结果或失败原因 |
| P1 常规任务 | 3.30 | 2.20 | 标准生产任务，24 小时内完成 |
| explore 探索任务 | 按 P1 计费 | 1.20 | 非登录态公开页面探索，不承诺全量 |
| P2 | 3.30 | 2.20 | 当前不启用独立价格档，统一按 P1 计费 |

说明：

1. P1 价格与原有结算口径保持一致：登录态 3.30 元 / 1000 页面，非登录态 2.20 元 / 1000 页面。
2. 任务优先级由 Anker 在提交任务时指定，服务方按 Anker 传入的优先级进行调度、统计和结算。
3. P0 适用于舆情危机、紧急差评核实、下架排查、投诉证据核实等高优先级任务。
4. explore 仅适用于非登录态公开页面探索，需由 Anker 在任务中明确指定；未指定 priority 时仍按 P1 结算。
5. 如 explore 任务要求登录态、翻页完整性或强 SLA，则按 P1 或 P0 结算。
6. P2 当前不作为独立优惠档；如接口传入 P2，结算时归入 P1。

### 3.3 评论采集按评论数量计费（可选）

评论采集任务可选择按“成功返回评论条数”计费。该口径仅适用于评论采集类任务，与页面量计费二选一，不重复收费。

计费单位：元 / 1000 条成功返回评论。

| 优先级 | 评论数量单价 | SLA / 说明 |
|---|---:|---|
| P0 紧急任务 | 0.80 | 4 小时内返回结果或失败原因 |
| P1 常规任务 | 0.50 | 24 小时内完成 |
| explore 探索任务 | 0.30 | 仅限非登录态公开页面探索 |
| P2 | 0.50 | 当前不启用独立价格档，统一按 P1 计费 |

说明：

1. 评论数量计费以当月成功返回给 Anker 的评论记录数为准。
2. 同一任务内重复的 `review_id` 不重复计费。
3. 系统内部重试、账号切换、验证码重试不计入评论数量费用。
4. 评论数量计费只适用于评论采集结果，不适用于商品详情、Listing、Search Rank、Top100 等页面类任务。
5. 同一批评论采集任务在月结时只能选择一种计费口径：按页面量计费或按评论数量计费。

计费公式：

```text
评论数量费用 = sum(成功返回评论条数 / 1000 * 对应优先级评论单价)
```

## 四、任务折算规则

| 任务类型 | 计费页面折算 |
|---|---:|
| 评论采集 | 默认按实际成功采集页面数计费；也可选择按成功返回评论条数计费 |
| 商品详情采集 | 按实际成功采集页面数计费 |
| Listing / Search Rank / Top100 | 按实际成功采集页面数计费 |
| Rufus / 特殊页面 | 按登录态成功页面计费 |
| 评论存在检查 | 成功检查数 * 0.5 页面 |
| 星级分布统计 | 成功 ASIN 数 * 1 页面 |
| 非登录态 Top Reviews explore | 成功 ASIN 数 * 1 页面 |
| 缓存复用结果 | 按对应单价的 20% 折算 |

计费公式：

```text
采集流量费 = sum(成功页面量 / 1000 * 对应单价)
评论数量费用 = sum(成功返回评论条数 / 1000 * 对应评论数量单价)
缓存复用费 = sum(复用页面量 / 1000 * 对应单价 * 20%)
```

## 五、基础服务费（固定月费）

基础服务费用于覆盖 Anker VOC 采集所需的基础资源和持续运维服务，包括采集算力池、代理池、Amazon 账号、数据存储、采集运维、监控告警和月度对账支持。

当前 Anker 月度量级约为 16,923,283 页面，折合约 564,109 页面/日、6.5 页面/秒的平均成功吞吐。考虑 Amazon 登录态采集、P0 抢占、重试、代理切换、账号池轮换和峰值任务，不能仅按固定 8 台低配服务器解释，需要按“采集算力池 + 代理池 + 账号池”的综合资源预估。

在当前服务范围和当前量级不变的情况下，基础服务费预估为 38,000.00 元/月，不随当月小幅采集量波动变化。若双方确认新增或减少采集算力池规模、代理池规模、Amazon 账号规模、专属并发、存储周期或维护范围，基础服务费需同步调整。

| 费用组成 | 预估金额（元/月） | 说明 |

| 采集运维与监控 | 6,000.00 | Rufus / 特殊页面维护、风控变化处理、任务异常排查、成功率和时效监控、月度对账支持 |

## 八、结算规则

1. 每月按自然月统计用量并出具结算单。
2. 页面类采集按成功页面量结算，评论采集可选择按成功页面量或成功返回评论条数结算，内部重试不重复计费。
3. 最终失败任务默认不收取采集流量费，但会进入失败率和 SLA 统计。
4. ASIN 不存在、参数错误、权限不足等无效任务不进入采集流量费。
5. 缓存复用结果按对应单价 20% 折算。
6. 回调失败但结果已成功采集且可查询时，采集流量费正常计算；回调失败记录进入对账明细。
7. 基础服务费为当前量级资源池预估固定月费，当前服务范围下为 38,000.00 元/月，不随小幅采集量波动自动调整。
8. 新增资源、额外开发、特殊 SLA 或超出本报价范围的需求，需双方确认后再调整。

## 十一、报价确认

如 Anker 确认本报价，按以下方式落地：

6. 每月结算单按业务方、任务类型、优先级、登录态拆分展示，双方据此核对。
