Vidaxl 数据量问题 · 决策

客户原话：「Vidaxl 全部站点只有 198-200，是我们只抓 200 还是对方限我们 200？」

先看诊断结论，再选 UI 方案 + 扩量档位。

⚠️ 诊断结论

100% 是我们这边设的上限，不是 Vidaxl 限制我们。证据：

• backend/sites.yaml Vidaxl 全 12 行写死 max_products: 200
• backend/app/crawlers/vidaxl.py:28 STOREFRONT_LIMIT = 200 默认值
• vidaxl.py:145 抓到 200 就 break

为什么当初设 200：Vidaxl 单站 ~25 万 SKU，全量爬几小时 + 大带宽 + 高风控风险；MVP 阶段验证通路够了。SONGMICS / Costway 没限（DE 11624 / Costway CA 1607）是因为 Shopify / API 路径一次性能拉完。

核心数据：Vidaxl 全 12 站平均覆盖率 ~0.08% · 升到 2K 档 = 10× · 升到全量 = 200×
