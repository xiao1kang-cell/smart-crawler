3.3 关键算法：sitemap lastmod 增量

# 大部分电商 sitemap.xml 有 lastmod 字段，是增量利器

# 例：vidaxl.de/sitemap_index.xml
<sitemap>
  <loc>https://www.vidaxl.de/sitemap_custom-products_1.xml</loc>
  <lastmod>2026-05-24T04:32:18+00:00</lastmod>   # ← 这个
</sitemap>

# 增量算法
def daily_delta(site, yesterday_ts):
    sitemaps = parse_sitemap_index(site.sitemap_url)

    # 1. 只取昨天后改过的 sitemap 文件（粗筛）
    fresh_maps = [m for m in sitemaps if m.lastmod > yesterday_ts]

    # 2. 再细化到 URL 级别
    fresh_urls = []
    for m in fresh_maps:
        urls = parse_sitemap(m.loc)
        fresh_urls.extend([u for u in urls if u.lastmod > yesterday_ts])

    # 3. 对照 DB 决定 INSERT 或 UPDATE
    existing_skus = db.query(Product.url).filter_by(site=site).all()
    new_urls = [u for u in fresh_urls if u.loc not in existing_skus]
    updated_urls = [u for u in fresh_urls if u.loc in existing_skus]

    # 4. 优先级抓取
    return {
        "insert": new_urls,     # 必须抓
        "update": updated_urls, # top 1000 才抓
    }
边界情况：
  • sitemap 没 lastmod → fallback 到 URL set diff（vs 昨天 sitemap 快照）
  • sitemap 不诚实（lastmod 永远是 today）→ 改用 product page 内 JSON-LD dateModified
  • 没 sitemap（如 Costway 等 SPA）→ 用 /api/products?sort=newest 拉前 N 页

4评论增量（VOC 每日）

遨森 RFP 要求 21 个评论平台。评论比 SKU 更"日增量友好"——只抓 review_date > 上次最新即可。

# 每日评论增量逻辑
def daily_review_delta(merchant, platform):
    # 1. 拿 DB 中该商家最新评论日期
    last_date = db.query(max(Review.review_date)).filter_by(
        platform=platform, merchant=merchant
    ).scalar()

    # 2. 抓评论页（按时间倒序），直到撞上 last_date
    page = 1
    new_reviews = []
    while True:
        reviews = scrape_review_page(platform, merchant, page=page)
        # 撞到上次最新就停
        if any(r.date <= last_date for r in reviews):
            break
        new_reviews.extend(reviews)
        page += 1
        if page > 10: break  # 防御上限

    # 3. 跑 NLP 抽 sentiment / topic
    for r in new_reviews:
        r.sentiment, r.nlp_topics = llm_analyze(r.content, r.language)

    db.bulk_insert(new_reviews)
    return len(new_reviews)

这是 smart-crawler 已有的 trustpilot.py / google_maps.py / reviews_io.py 模式，需要包装成 daily delta job。当前是 batch run，要改成 incremental。
