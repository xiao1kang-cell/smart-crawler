1Step 1 · 关掉 iMac 的 Wi-Fi MAC 随机化

不然每次连 Wi-Fi MAC 变，AT&T 路由路由不到 iMac。

iMac 上：

  • ① 系统设置 → Wi-Fi
  • ② 当前连接的 Wi-Fi 右边 ... 详细信息
  • ③ "私人 Wi-Fi 地址" 关闭 OR 选 "固定"
  • ④ 看到 MAC 变成真实的 2c:ca:16:4e:d7:4f 即成功
关掉后 iMac 会断 Wi-Fi 重连一次，IP 可能变。重连后再 SSH 验证。

2Step 2 · 进 AT&T modem 后台开 "Public Subnet" 模式

让 modem 把整个 /26 段路由到 iMac MAC（不要 NAT）。

路径（AT&T BGW210/BGW320/Pace 5268 通用）：

  • ① 浏览器开 http://192.168.1.254
  • ② 登录（密码在 modem 底标 "Device Access Code"）
  • ③ 顶部菜单 Settings → LAN → DHCP
  • ④ 找 "Public Subnet"（有的型号叫 "Cascaded Router" 或 "Supplementary Network"）
  • ⑤ 启用 Public Subnet
  • ⑥ 把 Subnet Range 填 108.95.61.128 / 255.255.255.192
  • ⑦ Gateway 填 108.95.61.190
  • ⑧ Allowed IPs: 全选 .129~.189
  • ⑨ Default Server 选 iMac 的 MAC 2c:ca:16:4e:d7:4f（或 IP 192.168.1.91）
  • ⑩ Save · modem 自动重启
如果 modem 不支持 Public Subnet：换 "IP Passthrough" 模式，但只能给 iMac 1 个公网 IP。或者打 AT&T 客服开 "Cascaded Router" 模式 → 给整个 /26。

3Step 3 · iMac 上配 61 个 IP alias

modem 重启后，iMac 应该能拿到 /26 段一个 IP。然后手动加另外 60 个 alias。

iMac 上跑这个 (Terminal)：

# 取消之前 192.168.1.91 的 DHCP，改静态拿一个 /26 IP
# 先验证 modem 配好了 - 看 iMac 能不能拿到公网 IP
ifconfig en0 | grep "inet "
# 应该看到 108.95.61.x 这样的 IP（如果还是 192.168.1.91，说明 modem 没配好）

# 加 60 个 alias（假设第 1 个 IP 已经是 .129）
for i in {130..189}; do
    sudo ifconfig en0 alias 108.95.61.$i 255.255.255.192
done

# 验证：应该看到 61 个 inet 行
ifconfig en0 | grep -c "inet "

# 测每个出口 IP 是否真生效（用其中一个 IP 出去打 ipify）
curl --interface 108.95.61.150 https://api.ipify.org
# 应该返回 108.95.61.150（说明这个 IP 真的能出 internet）
持久化：上面 ifconfig alias 重启 iMac 后会丢。要 launchd 持久化。我装 brew 后给你装一个 plist。

⚠️常见坑

  • modem 不在 Public Subnet 选项 → 打 800-288-2020 让 AT&T 帮开 "Cascaded Router" 或 "Routed Public Subnet"
  • iMac 还是私网 IP → modem 配错或没重启。重启 modem（拔电 30 秒）再试
  • alias 加了但 curl --interface 失败 → modem 还在做 SNAT，公网 IP 被改写。检查 modem 的 "NAT/Firewall" 设置改 Pass-Through
  • iMac 睡眠后断网 → 系统设置 → 节能 → "防止电脑自动睡眠"
