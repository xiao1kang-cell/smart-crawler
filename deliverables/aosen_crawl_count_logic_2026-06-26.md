# 标杆商品抓取计数逻辑说明

> 适用场景：解释后台“已抓商品数 / 覆盖率 / 字段完整率”的统计口径  
> 更新时间：2026-06-26

## 1. 商品抓取数统计的是什么

后台展示的“已抓商品数”，统计的是系统已经识别并入库的商品记录。

系统会从多个入口发现商品：

- 标杆网站的商品列表页
- 站点 sitemap
- 分类页
- 商品详情页
- 部分站点开放的商品接口

只要系统确认某个 URL 或商品编号对应的是有效商品，并成功生成商品记录，就会计入“已抓商品数”。

因此，“已抓商品数”主要回答的是：

> 这个站点目前已经有多少商品被系统识别并进入商品库。

## 2. 商品计数和字段完整性是两个指标

商品入库后，系统还会继续校验每个商品的字段是否完整，例如：

- 商品标题
- 商品图片
- 商品价格
- 原价 / 折扣价
- 币种
- 类目
- 库存状态
- 评分和评论数
- 促销信息

所以商品数据会分成两层来看：

| 指标 | 含义 |
|---|---|
| 已抓商品数 | 商品是否已被识别并入库 |
| 字段完整率 | 入库商品的价格、标题、图片、类目等字段是否完整 |

也就是说，商品已经计入“已抓商品数”，不代表它的所有字段都已经完整。

例如：某个商品已经识别到 URL、标题和图片，并进入商品库，但价格字段暂时没有解析到。这个商品会计入“已抓商品数”，同时会在数据质量检查中被标记为“价格缺失”。

## 3. 覆盖率的统计逻辑

覆盖率用于衡量当前已抓商品量与目标商品量之间的比例。

简化公式如下：

```text
覆盖率 = 当前已抓商品量 / 目标商品量
```

目标商品量通常来自以下来源：

- 站点 sitemap 中的商品 URL 数量
- 标杆网站页面可见的商品数量
- 系统历史抓取和校准后的目标值
- 人工配置的目标 SKU 数

如果一个站点的 sitemap 或页面结构发生变化，目标商品量也会随之校准。

因此，覆盖率主要回答的是：

> 当前系统抓到的商品规模，距离该站点应抓商品规模还有多远。

## 4. 为什么会出现“商品已抓到，但价格缺失”

不同网站的价格展示方式不一样。有些网站在列表页就能拿到价格，有些网站必须进入商品详情页，有些网站的价格由前端接口动态加载。

因此，系统会先保证商品被识别并入库，再对价格等字段做补充解析和质量校验。

这会出现一种正常的中间状态：

> 商品已经进入商品库，但价格字段还需要继续补采或校验。

这类商品不会被当作“价格完整商品”，而是会在数据质量页中单独标记为“价格缺失”，并进入后续补采范围。

## 5. 客户验收建议口径

建议将数据验收拆成三层：

| 验收层级 | 验收内容 |
|---|---|
| 商品覆盖 | 看站点是否已经抓到足够的商品规模 |
| 字段完整 | 看标题、图片、价格、类目、库存等字段是否完整 |
| 业务分析 | 看价格趋势、促销、销量估算、营收估算等分析结果是否可用 |

这三层应该分开验收：

- 商品覆盖达标，说明商品库规模已经建立；
- 字段完整达标，说明商品数据可以进入业务使用；
- 业务分析达标，说明数据已经可以支持趋势、促销、销售等分析。

## 6. 对当前情况的说明

当前后台中的商品数量，表示系统已经识别并入库的商品记录。

对于部分站点，商品基础信息已经入库，但价格字段仍有缺失。这类问题不会影响商品数量统计，但会影响价格完整率和价格分析结果。

我们会按数据质量页继续跟进价格、标题、图片、类目等字段的完整性，并在字段补齐后再推进最终验收。
