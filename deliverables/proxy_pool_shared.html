1AT&T Modem · Cascaded Router 模式

把整个 /26 段路由给 iMac 接管 — 不要 Public Subnet 模式（那个 modem 只学单个 ARP）

# 浏览器开
http://192.168.1.254

# 登录密码：modem 底标 "Device Access Code"

# 顶部菜单 Network → Subnets & DHCP

# Public Subnet 一段：
Public Subnet Mode = Off     # 改成 Off

# Cascaded Router 一段：
Cascaded Router          = Enable
Cascaded Router Address  = 192.168.1.91     # iMac 私网 IP
Network Address          = 108.95.61.128
Subnet Mask              = 255.255.255.192

# Save · modem 立刻生效（不一定重启）
原理：Cascaded Router 模式下，modem 不再要求 ARP 学 /26 内每个 IP，而是直接转发整段给 iMac。iMac 自己接管。Public Subnet 模式下 modem 只学第一个分配的 IP，其他 alias 包被静默丢弃。
万一 modem 没有 Cascaded Router 选项：打 800-288-2020，让 AT&T 客服后台把 modem 的 routing mode 设成 "Routed Static Subnet to LAN device"。

2iMac · 50 个 IP alias（持久化）

modem 转完，iMac 自己接管整段。一次性配 50 个 alias + launchd 开机自启

2.1 临时配（测试用，重启会丢）

# Terminal on iMac (192.168.1.91)
# 关闭 Wi-Fi MAC 随机化先（在系统设置里），然后：
for i in {130..189}; do
  sudo ifconfig en0 alias 108.95.61.$i 255.255.255.192
done

# 验证：应该看到 51 个 inet 行（原本 1 个 + 50 alias）
ifconfig en0 | grep -c "inet "

# 测每个出口 IP
for i in {130..189}; do
  echo "$i:" $(curl --interface 108.95.61.$i -s --max-time 5 https://api.ipify.org)
done

3multi_ip_proxy · 50 端口 HTTP 代理

每个端口 bind 一个出口 IP，统一 Basic Auth · aiohttp 单文件 · ~50MB RAM

3.1 multi_ip_proxy.py

# ~/multi_ip_proxy.py · 单文件，零依赖（用 stdlib + aiohttp）
import asyncio, os, sys, ssl, base64
from aiohttp import web, ClientSession, TCPConnector

def make_app(egress_ip, listen_port, auth_user, auth_pass):
    async def handle(req):
        # Basic Auth 校验
        h = req.headers.get("Proxy-Authorization", "")
        expected = "Basic " + base64.b64encode(
            f"{auth_user}:{auth_pass}".encode()).decode()
        if h != expected:
            return web.Response(status=407,
                headers={"Proxy-Authenticate": "Basic realm=proxy"})

        # Forward via egress_ip
        connector = TCPConnector(local_addr=(egress_ip, 0))
        async with ClientSession(connector=connector) as s:
            async with s.request(req.method, str(req.url),
                    headers={k:v for k,v in req.headers.items()
                             if k.lower() != "proxy-authorization"},
                    data=await req.read(), allow_redirects=False) as r:
                body = await r.read()
                return web.Response(body=body, status=r.status,
                    headers={k:v for k,v in r.headers.items()
                             if k.lower() not in ("transfer-encoding","content-encoding")})

    app = web.Application()
    app.router.add_route("*", "/{path:.*}", handle)
    runner = web.AppRunner(app)
    return runner

async def main():
    USER = os.environ.get("PROXY_USER", "shared")
    PASS = os.environ["PROXY_PASS"]   # 强制 env 配密码
    runners = []
    for i in range(130, 190):       # 60 个 IP
        egress = f"108.95.61.{i}"
        port = 8000 + (i - 129)         # 8001..8060
        runner = make_app(egress, port, USER, PASS)
        await runner.setup()
        site = web.TCPSite(runner, "0.0.0.0", port)
        await site.start()
        runners.append(runner)
        print(f"listening :{port} → egress {egress}")
    await asyncio.Event().wait()       # 永久 block

if __name__ == "__main__":
    asyncio.run(main())

4共享访问 · 团队跨项目用法

公司任意服务器 / 同事 Mac / 任意项目，3 种典型用法

4.1 LAN 内（公司局域网设备）

# 直接 HTTP_PROXY env var，每个 shell session 或 cron job 都行
export HTTP_PROXY=http://shared:<pwd>@192.168.1.91:8001
export HTTPS_PROXY=http://shared:<pwd>@192.168.1.91:8001

# 验证
curl https://api.ipify.org   # 应该返回 108.95.61.130

# 随机切到另一个出口 IP
export HTTP_PROXY=http://shared:<pwd>@192.168.1.91:8050  # 端口 8050 = .179 IP

4.2 远程团队（Tailscale 推荐）

# iMac 上装 Tailscale，加入公司 tailnet
brew install tailscale && sudo tailscale up

# 同事在他自己设备上装 Tailscale 加入同一 tailnet
# 然后用 iMac 的 100.x.x.x Tailscale IP（不是 192.168.1.91）
export HTTP_PROXY=http://shared:<pwd>@100.x.x.x:8001

# Tailscale 上密码鉴权 + ACL 双重防护，无需开公网端口

4.3 Python / requests / curl 一键脚本

# Python 随机轮询
import random, requests
PROXY_HOST = "192.168.1.91"   # 或 Tailscale IP
PROXY_USER, PROXY_PASS = "shared", "<pwd>"

def random_proxy():
    port = random.randint(8001, 8050)
    return f"http://{PROXY_USER}:{PROXY_PASS}@{PROXY_HOST}:{port}"

p = random_proxy()
r = requests.get("https://api.ipify.org", proxies={"http":p,"https":p})
print(r.text)   # 每次不同 IP

5权限 / 安全

层机制谁能进
L1 网络iMac 防火墙：8001-8050 仅放给 LAN (192.168.1.0/24) + Tailscale (100.64.0.0/10)办公室同事 + Tailscale 已加入设备
L2 鉴权HTTP Basic Auth: shared:<pwd>知道密码的人
L3 审计/var/log/multi-ip-proxy.log 记每条请求 (target_host + egress_ip)事后追溯
密码轮换：改 launchd plist 的 PROXY_PASS → sudo launchctl unload + load → 通知所有项目改 URL。建议季度轮换。

6健康监控 · 自动 failover

6.1 NAS 上跑健康检查 cron

# /usr/local/bin/proxy_health.sh · 每 5 分钟跑一次
# 测每个端口的 egress IP，挂掉的标记 down

for port in {8001..8050}; do
  ip=$(curl -s --max-time 5 \
    -x http://shared:$PASS@192.168.1.91:$port \
    https://api.ipify.org)
  if [[ -z "$ip" ]]; then
    echo "DOWN port=$port"
  else
    echo "OK   port=$port → $ip"
  fi
done > /var/log/proxy_health.log

6.2 smart-crawler 已内置 failover

每个 proxy URL 失败 3 次自动剔除 10 分钟（PROXY_FAIL_THRESHOLD=3 / PROXY_COOLDOWN_SEC=600）。dashboard /api/proxy/status 实时看每个 IP 的 success/fail。
